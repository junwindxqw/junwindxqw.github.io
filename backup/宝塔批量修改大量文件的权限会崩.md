最近网站被攻击，为了加强安全，使用宝塔将public目录下uploads目录递归修改权限为700。
结果宝塔死了。

为什么面板会“崩”？
PHP 执行超时（Timeout）： 宝塔面板的后端主要是靠 PHP 和 Python 脚本运行的。当你点击“应用到子目录”时，系统会遍历每一个文件。如果文件数量达到几万甚至几十万个，处理时间会超过面板预设的脚本执行限制，导致前端页面失去响应或报错。

I/O 负载过高： 修改权限是一个密集的磁盘读写操作。大量小文件的高频修改会导致磁盘 I/O 瞬间飙升到 100%，此时面板的数据库读取、日志写入都会被阻塞，导致你访问面板时出现“无法连接”或响应极慢。

系统资源耗尽： 如果服务器配置（尤其是 CPU 和内存）较低，大规模的递归遍历会占用大量系统资源，导致宝塔的面板服务（bt 守护进程）因资源不足而重启或挂起。

反思：对大量文件做批量修改，不要直接用宝塔处理。
